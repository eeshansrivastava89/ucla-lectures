"It works on my machine" is not a deployment strategy. <span class="accent">Containers solve this: same code, same dependencies, same behavior on your laptop, your teammate's laptop, and the server.</span> This lecture wraps the prediction service and MLflow in Docker, then adds Prefect so the system retrains on a schedule.

A Dockerfile is a short script that describes exactly what your service needs to run — Python version, packages, environment variables. Docker Compose orchestrates multiple containers so the API, MLflow, and Prefect all start together with one command. Prefect adds scheduled retraining: instead of manually rerunning a notebook when you want a fresh model, the pipeline runs it automatically, logs the results, and registers the new version.

By the end of this lecture your service is reproducible. <span class="accent">Anyone on your team can run it. It retrains without you.</span> That is the difference between a script and a system.
