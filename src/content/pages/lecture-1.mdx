Every ML project starts in a notebook. A notebook is a fine place to explore data and train a model. <span class="accent">It is not a product.</span> It has no API, no error handling, no way for anything else to call it.

This lecture takes a trained churn model and deploys it as a live prediction endpoint. You write a FastAPI app, define a `/predict` route, and serve predictions over HTTP. This is the first step from prototype to system, and it is the shortest step in the course. The rest of the lectures are about whether you can trust what this endpoint tells you.

Because deploying the model is the easy part. The harder question is what happens when it returns a wrong prediction. <span class="accent">Every ML component makes mistakes. That is normal.</span> The question is whether the system around it handles those mistakes or quietly breaks. You start answering that question here, and you spend the rest of the course building the answer.
