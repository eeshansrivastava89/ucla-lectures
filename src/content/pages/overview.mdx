This is a course for students who want to **build software products with machine learning, not just models**. We assume you can train a model in Python. The question this course answers: <span class="accent">what does it take to turn that model into a product</span> — to deploy it, have confidence in its quality, and operate it in the real world?

The course covers the full lifecycle: from a notebook prototype to a deployed prediction API, through experiment tracking, containerization, monitoring, and finally wrapping the system with an LLM-powered conversational interface. Each lecture adds one production capability to the same codebase. By the end of five weeks you have built a working AI product end-to-end.

We also cover what happens when things go wrong — <span class="accent">because they will</span>. Every ML component makes mistakes. The question is whether the system around it handles those mistakes or quietly breaks. We address safety, fairness, security, and the operational reality of maintaining ML systems at scale. The course closes by mapping every FOSS tool you used to its enterprise counterpart, so you know exactly what changes when the project goes from your laptop to a production environment serving millions of users.

## Who This Course Is For

You have trained ML models in Python. You know scikit-learn, pandas, and Jupyter notebooks. You may have built models for class projects or at work, <span class="accent">but you have not had to deploy them, monitor them in production, or build the infrastructure around them</span>.

This course does not assume a software engineering background. You do not need experience with Docker, CI/CD, or cloud infrastructure. We teach those as we go. You do need basic Python fluency, comfort with the command line, and willingness to read documentation and learn new tools on your own.

If you are unsure whether your background is sufficient, look at the Lecture 1 deliverable: <span class="accent">deploy a FastAPI endpoint that serves predictions from a trained model</span>. If you can train a model in Python, you can do this.

## Topics Covered

The course addresses the following questions, organized by lecture:

- **From notebook to production service.** <span class="accent">How do you take a model that works in a notebook and deploy it as a live API?</span> What happens when the endpoint returns a wrong prediction, and how should the system handle that?
- **Experiment tracking and model selection.** How do you track experiments so your work is reproducible and auditable? <span class="accent">Which qualities matter beyond accuracy</span> — latency, cost, model size, fairness — and how do you navigate tradeoffs between them?
- **Infrastructure and orchestration.** How do you make a service reproducible across machines? How do you automate retraining so the model stays fresh without manual intervention?
- **Monitoring, testing, and CI/CD.** How do you detect when a model's performance degrades in production? How do you distinguish data quality problems from drift from feedback loops? How do you audit for bias and run A/B tests with statistical rigor?
- **LLM interface and safety.** How do you wrap a prediction API with an LLM so raw scores become explanations and suggested actions? <span class="accent">How do you handle hallucination, prompt injection, and adversarial inputs?</span> How do you connect ML output to business decisions?

## Examples and Case Studies

The running example for this course is customer churn prediction: a supervised ML problem that is universally understood in business contexts. Students build a system that predicts which customers are likely to leave, explains why in plain English, and suggests retention actions.

The course architecture is domain-agnostic. The same pipeline works for fraud detection (flag suspicious transactions, explain risk scores), demand forecasting (predict product demand, answer driver questions), or recommendation systems (recommend items, explain why). Students may substitute any domain with a public dataset and a natural "ask questions about the results" story.

Throughout the course we reference real production systems and case studies: how Spotify personalizes playlists, how automated transcription services handle noisy audio at scale, how fraud detection systems balance false positives and customer friction.

## Course Project

Over 5 weeks you build one project that grows each week. You start with a trained churn model in a notebook. <span class="accent">Each lecture adds a new capability to the same codebase:</span>

1. Deploy the model as a FastAPI endpoint. Test it with real requests and edge cases.
2. Track experiments with MLflow. Compare models across multiple quality dimensions. Register the best model.
3. Containerize the service with Docker. Add Prefect for scheduled retraining.
4. Add pytest tests, Evidently monitoring, bias audits, and GitHub Actions CI/CD.
5. Wrap the API with an LLM interface. Add safety guardrails. Ship the full product.

You submit the same repository each week, showing additive progress. The final deliverable is a working AI product — not a final exam.

## Learning Outcomes

After taking this course, you should be able to:

- Deploy a trained ML model as a production API with error handling for malformed input
- Track, compare, and register models using experiment tracking tools
- Justify model selection based on multiple quality dimensions beyond accuracy
- Containerize an ML service and automate retraining with a pipeline orchestrator
- Detect data quality issues, data drift, concept drift, and feedback loops in production
- Audit ML models for bias across user segments
- Design and run A/B tests to compare model versions with statistical rigor
- Build a CI/CD pipeline that validates code and models on every push
- Wrap a prediction API with an LLM to produce explanations and actionable insights
- Implement guardrails for hallucination, prompt injection, and adversarial inputs
- Map FOSS MLOps tools to their enterprise counterparts and understand what changes at each level of scale
